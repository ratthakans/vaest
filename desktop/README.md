# VÆST — desktop shell (Tauri v2)

A thin native window that loads the live app (`https://vaest.orions.agency`). Because it
always loads the current site, **the content updates itself** — you never rebuild the app to
ship a UI change. The native shell only needs a new build for shell-level changes (window,
menu, updater), and those go out through the auto-updater.

Not a rewrite of the web app — the web app is untouched. This folder is self-contained.

## Layout

```
desktop/
  package.json            # @tauri-apps/cli + npm scripts
  shell/index.html        # tiny on-brand loading page → redirects to the live site
  src-tauri/
    tauri.conf.json       # window, bundle (dmg/app), updater config
    Cargo.toml            # Rust deps
    build.rs
    src/main.rs, lib.rs   # minimal Tauri entry
    capabilities/         # v2 permissions
    icons/                # generated by `tauri icon`
```

## Build (unsigned — for testing)

Prereqs: **Node**, **Xcode Command Line Tools**, and **Rust** (`https://rustup.rs`).

```bash
cd desktop
npm install
npm run build        # → src-tauri/target/release/bundle/dmg/VÆST_<ver>_aarch64.dmg
```

The unsigned `.dmg` opens on your own Mac, but on another machine Gatekeeper will say
"unidentified developer." Right-click → Open works; for real distribution, sign + notarize
(below).

Dev (hot window, no packaging): `npm run dev`.

## Signing + notarization (for distribution) — needs an Apple Developer account ($99/yr)

Not required to test locally. When ready:

1. Enroll in the Apple Developer Program, create a **Developer ID Application** certificate,
   install it in your login keychain.
2. Create an app-specific password for `notarytool` (appleid.apple.com → Sign-In & Security).
3. Set env vars, then build:
   ```bash
   export APPLE_SIGNING_IDENTITY="Developer ID Application: Your Name (TEAMID)"
   export APPLE_ID="you@icloud.com"
   export APPLE_PASSWORD="app-specific-password"
   export APPLE_TEAM_ID="TEAMID"
   npm run build
   ```
   Tauri signs and submits to Apple automatically when these are present.

These credentials are yours — keep them out of the repo (they're in `.gitignore`-safe env,
never committed).

## Auto-update (wire once you have a place to host the release feed)

Tauri's updater checks a small JSON feed and, if a newer signed build exists, downloads and
installs it. Two one-time setup steps:

1. **Updater keypair** (Tauri's own signing for update integrity — separate from Apple):
   ```bash
   npx tauri signer generate -w vaest-updater.key   # keep the PRIVATE key secret, never commit
   ```
   Put the printed **public** key in `src-tauri/tauri.conf.json` → `plugins.updater.pubkey`.
2. **Release feed**: host `latest.json` at the `endpoints` URL (e.g. on Vercel at
   `/desktop/latest.json`). On each release, `npm run build` emits a signed
   `.dmg` + a `.sig`; publish the `.dmg` somewhere public and point `latest.json` at it:
   ```json
   {
     "version": "3.0.1",
     "notes": "What changed",
     "pub_date": "2026-07-18T00:00:00Z",
     "platforms": {
       "darwin-aarch64": { "signature": "<contents of the .sig>", "url": "https://.../VÆST_3.0.1_aarch64.dmg" },
       "darwin-x86_64":  { "signature": "<...>", "url": "https://.../VÆST_3.0.1_x64.dmg" }
     }
   }
   ```
   The app checks this on launch; a newer `version` triggers the update.

Until the feed exists, the updater simply finds nothing and the app runs normally.
